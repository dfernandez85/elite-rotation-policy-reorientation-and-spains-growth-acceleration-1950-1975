# Elite Rotation, Policy Reorientation, and Spain's Growth Acceleration, 1950-1975

This repository contains the replication package for the article *Elite Rotation, Policy Reorientation, and Spain's Growth Acceleration, 1950-1975*. The project reproduces the econometric results reported in the main manuscript and the online appendix through a single synthetic-control pipeline. The current version is structured as a journal-ready replication package and exports only the final figures and tables cited in the paper.

## Quick start
From R:

```r
install.packages("renv")  # only if renv is not yet installed
renv::restore()
source("main.R")
```

From the terminal:

```powershell
Rscript main.R
```

The first run requires an internet connection. If the pinned raw inputs are not present in `data/raw/`, `main.R` downloads them automatically and refreshes `data/raw/checksums.txt`.

## Repository structure
```text
R/
  config/         # Constants, package loading, and pipeline settings
  data/           # Data acquisition and preprocessing
  analysis/       # Estimation pipeline and final journal exports
  utils/          # General helper functions
  visualization/  # Plotting functions
scripts/
  fetch_data.R    # Forces a clean download of pinned raw inputs
output/
  sessions/<timestamp>/
    plots/
      main/       # Figures for the main manuscript
      appendix/   # Figures for the online appendix
    tables/
      main/       # Tables for the main manuscript
      appendix/   # Tables for the online appendix
    run_config.yml
    session_info.txt
main.R            # Main entry point
```

## Dependencies
- R 4.x
- `renv`: run `renv::restore()` before executing `main.R`
- The packages listed in `R/config/packages.R`, loaded internally through `load_required_packages()`

## Data and reproducibility
- `main.R` is self-bootstrapping: if the required raw inputs are missing or their checksums do not match, the script downloads the pinned source files and rewrites `data/raw/checksums.txt`.
- The repository does not version raw binary data or generated outputs. Raw inputs are recovered from pinned upstream sources, and all results are regenerated by running `main.R`.
- Required raw files:
  - `data/raw/pwt110.xlsx` (Penn World Table 11.0, `Data` sheet), checksum key `pwt110`
  - `data/raw/maddison_mpd2023.dta` (Maddison Project Database 2023), checksum key `maddison_mpd2023`
- Checksums are stored in `data/raw/checksums.txt` using the format `<key> <md5sum>`.
- To force a clean re-download of the pinned inputs, run `Rscript scripts/fetch_data.R`.
- The repository includes a GitHub Actions workflow at `/.github/workflows/reproducibility-check.yml` that verifies a clean Windows run of `renv::restore()` and the full execution of `main.R`, and uploads the resulting output session as an artifact.

## Running the pipeline
```r
renv::restore()        # once per environment
source("main.R")       # from R / RStudio
# or
Rscript main.R         # from the terminal
```

After `renv::restore()`, no manual preparation of `data/raw/` is required; the pipeline bootstraps those inputs automatically when needed.

Each execution creates a new timestamped folder under `output/sessions/<timestamp>/`. The final visible output is restricted to `plots/main`, `plots/appendix`, `tables/main`, `tables/appendix`, plus `run_config.yml` and `session_info.txt`.

## Exported outputs
- `plots/main`: figures used in the main manuscript
- `plots/appendix`: figures used in the online appendix
- `tables/main`: tables used in the main manuscript
- `tables/appendix`: tables used in the online appendix
- `run_config.yml` and `session_info.txt`: reproducibility metadata for each session

## Module overview
- `R/analysis/synthetic_control.R`: prepares the panel, runs MSCMT, computes placebo exercises, fit filters, the stability gate, and the robustness specifications required by the paper
- `R/analysis/journal_exports.R`: takes the estimated objects and exports only the canonical figures and tables used in the manuscript and appendix; intermediate plot and table trees are removed at the end of the run so that each session remains clean

The authoritative, reproducible entry point for the project is `main.R`.
